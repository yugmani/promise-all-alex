  • What is Promise?

    1. Promises are objects that represent some value. The value is not determined at the time of creating a promise. But this value will be determined in the future when a promise is resolved or rejected.
    2. Promises make writing asynchronous code much easier and more manageable.
    3. For example: When our application needs to fetch data from some API, we don't know when will we receive this data. Sometimes, we get them immediately and sometimes it might take more time. It is not convenient to check the data in specific intervals nor keep waiting for long time. Using Promise is the solution to overcome from this problem. When Promise is settled ( resolved or rejected ), it automatically triggers events that we can listen to with specific handler methods and our app gets updated.

  • Four States of JavaScript Promises

    1. Pending: The first state. This is the initial state when we create a Promise and invoke it. It is the state of Promise that is neighter fulfilled(resolved) nor rejected.
    2. Fulfilled: This is the second state of Promise being successfully resolved.
    3. Rejected: This is the third state of Promise being rejected due to some problem that prevented the Promise from being successfully fulfilled.
    4. Settled: The last state that the job of Promise is finished and the Promise was either fulfilled or rejected.

  • Handling JavaScript Promises with Handler Functions

    1. The then(), catch() and finally() are handler functions you can attach to a Promise.
    2. When you invoke a Promise and the promise is settled(resolved or rejected) one of these handlers will be automatically invoked. When there are some data returned from a Promise it is passed into these handlers.
    3. We could put logic for updating our app with the data we received from API into these handlers. Without these handlers, the Promise would still run after we invoke it. These handlers are like messengers that transport the message from Promise further down the chain.

  • The then() handler function

    1. The then() handler is usually used to handle fulfilled state of JavaScript Promises.
    2. We need to attach it after the Promise when we invoke it.
    3. To access the data passed into the handler inside the Promise, we need to pass a callback function to the then() handler. This callback function should accept one parameter that represents the data returned by the Promise.

  • The catch() handler function

    1. The catch() handler is for handling rejected states of JavaScript Promises.
    2. It is a good practice to attach the catch() handler to a promise after the then() handler.
    3. While attaching it to a promise, we pass in a callback function that accepts on parameter. When Promise gets rejected any data passed into the reject() handler inside the Promise will be available through parameter.

  • The finally() handler function

    1. The last special handler function that will be invoked every time the Promise is settled.
    2. It will be invoked whether the Promise is fulfilled or rejected.
    3. It is useful when we want to do something regardless of the final state of a Promise.
    4. Unlike the then() and catch(), this handler doesn't require any parameter because there is nothing passed into it.

  • Promise.all()

    1. When we pass Promises into Promise.all() it will try to resolve all of them.
    2. When all Promises we passed are resolved Promise.all() will return one Promise that contains all values.
    3. We can then access this value by attaching then() handler to the Promise.all(), along with callback function.
    4. If one of the Promises gets rejected the Promises.all() will immediately return the rejected value. If one Promise 'fails' Promise.all() will return only the rejected value and does not return data from any previously resolved Promise(s).
    5. Note: Make sure to add catch() handler when you use Promise.all(). Or, add a second callback to then(). Otherwise, you will not get any error data if any Promise gets rejected.

  • Promise.allSettled()

    1. The Promise.allSettled() is another method we can use to handle multiple Promises.
    2. The Promise.allSettled works similarly to the Promise.all(). It als tries to resolve all Promises we pass into it.
    3. The main advantage on this is if any Promise is rejected the Promise.allSettled() waits for other Promises. So we don't have to worry about losing resolved values just because one Promise fails. We get all values from those Promises that are resolved as well as the rejected ones.
    4. This returns all values when all Promises are settled whether the Promises gets rejected or resolved.
    5. The values returned by Promise.allSettled() are in the form of an objects. Each object contains the status of the Promises. If Promise is fulfilled the corresponding object contains a value received from that Promise . If any Promise is rejected, the corresponding object contains reason with error data.

  • Promise.all()

    1. It takes a couple of Promises and let them race. It will return new Promise when one of the Promises that we passed into it fulfills or rejects as first.
    2. This new Promise contains either value if resolved or reason if rejected.
    3. Note Similarly to Promise.all(), also add catch() handler when you use Promise.race(). Or, add a second callback to then(). Otherwise, you will not get any error data if the first Promise gets rejected.

  • Promise.any()

    1. The Promise.any() is similar to Promise.race().
    2. It returns the Promise that is first to resolve(fulfilled).
    3. This ignores any Promise that is settled first but rejected.
